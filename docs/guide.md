# [Project] NLP 기반 중소기업 정책자금 맞춤형 추천 시스템 구축
> **목표:** 동적 웹 데이터 수집부터 LDA 토픽 모델링을 통한 지능형 추천 엔진 구현

---

## 1. Data Acquisition: 효율적 데이터 수집 (Reverse Engineering)
단순한 HTML 크롤링이 아닌, 웹 서비스의 백엔드 API 통신 구조를 분석하여 데이터를 획득했습니다.

* **배경:** 동적 로딩 페이지의 한계를 극복하기 위해 `POST` 요청 구조 파악
* **핵심 로직:**
    * `requests.post()`를 이용해 서버에 직접 `payload` 전송
    * `response.json()`을 활용해 정형 데이터(JSON) 즉시 획득 및 `pandas` 구조화
* **성과:** 대량의 공고 데이터를 유실 없이 가장 빠른 속도로 수집 가능

---

## 2. Text Preprocessing: 데이터 정제 및 형태소 분석
비정형 텍스트 데이터에서 분석 가치가 있는 '핵심 키워드'를 추출하는 과정입니다.

* **정규 표현식(re):** 공고 제목의 특수문자, 연도, 불필요한 태그(예: `[공고]`, `(수정)`) 제거
* **KoNLPy (Okt):** 한국어 특성을 고려한 형태소 분석 실시
    * `okt.nouns()`를 활용해 정책의 핵심 대상과 목적이 담긴 **명사** 위주로 데이터 필터링
* **빈도 분석:** `Counter`를 활용해 당해 연도 정책 자금의 주요 트렌드(예: '디지털', '수출', 'AI') 파악

---

## 3. Modeling & Recommendation: LDA 토픽 모델링 기반 추천
수집된 공고들을 잠재적인 주제(Topic)로 분류하고, 사용자 맞춤형 결과를 도출합니다.

### 🔹 LDA (Latent Dirichlet Allocation) 학습
* **Dictionary & BoW:** `gensim`을 이용해 단어 사전 구축 및 문서를 수치형(Vector)으로 변환
* **Topic Modeling:** `LdaModel`을 통해 수천 개의 공고를 'R&D', '수출지원', '인력양성' 등의 잠재 토픽으로 그룹화

### 🔹 맞춤형 추천 로직 (Logic)
1.  **사용자 쿼리 분석:** 사용자가 입력한 관심 키워드를 학습된 LDA 모델에 입력
2.  **토픽 매칭:** `get_document_topics()`를 통해 사용자의 관심사가 어떤 토픽에 속하는지 확률 계산
3.  **최종 필터링:** 해당 토픽 내에서 **최신순(`sort_values`)**으로 정렬하여 가장 적합한 정책 자금 추천

---

## 4. 확장 시나리오: 매출채권 분석을 통한 미래 현금흐름 예측
본 시스템은 단순 추천을 넘어, 소상공인 및 중소기업의 **정산 패턴 분석**으로 확장 가능합니다.

* **정산 패턴 파악:** 배달앱(월·수·금), POS(D+2), 이커머스 등 플랫폼별 입금 주기 DB화
* **매출채권 추정:** '정산 예정액'을 단기 매출채권으로 간주하여 미래 유입 예상 금액 시각화
* **기대효과:** 정책 자금 추천과 기업의 현금 흐름 분석을 결합한 종합 경영 컨설팅 대시보드 구현
